Role: Solution Architect Manager
This role leads the development of the system's software architecture, which includes promoting and creating support for the key technical decisions that constrain the overall design and implementation for the project.
Relationships
Additionally Performs
  • Perform Technical Review on Solution Architecture
Main Description

The software architect has overall responsibility for driving the major technical decisions, expressed as the software architecture. This typically includes identifying and documenting the architecturally significant aspects of the system as follows:

 

1.       Developing the design and architecture models for software products and product components.

2.       Analyzing and translating business, information and technical requirements into an architectural blueprint that outlines solutions that integrate across applications, systems, and platforms to achieve business objectives.

3.      Translate complex concepts into multiple architectural “views” aimed at varying audiences.

4.      Determine solution architectural requirements of the applications and services landscape, inline with technology strategy and application architect.

5.      Defines repeatable rules for how to implement solutions in more repeatable or reusable ways.

6.      Define the technical requirements, principles and models that guide all solution decisions.

7.    Define the reusable components needed and suggested components to be reusable. 

8.   Define system components, tools and technologies, deployment model and high-level specifications
