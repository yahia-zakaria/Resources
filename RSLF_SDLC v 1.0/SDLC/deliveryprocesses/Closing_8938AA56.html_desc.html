Description

Summary

 

This phase is designed to ensure that software is available for its users. The Deliver Phase can span several iterations, and includes testing the product in preparation for release, and making minor adjustments based on the feedback. At this point in the lifecycle, user feedback should focus mainly on fine tuning the product, configuring, installing and usability issues, all the major structural issues should have been worked out much earlier in the project lifecycle.

 

Objectives

 

Objective 1             

Achieving the pre-production testing to validate the new system against user expectations.

 

Objective 2             

Tuning activities such as bug fixing, enhancement for performance and usability.

 

Objective 3             

Achieving stakeholder concurrence that deployment baselines are complete.

 

Objective 4            

Achieving stakeholder concurrence that deployment baselines are consistent with the evaluation criteria of the vision.

 

Milestone

 

The Deliver phase is the fourth project milestone, the Product Release Milestone. At this point, Project team decides if the objectives were met, and if they should start another development cycle. The primary evaluation criteria for the Deliver phase involve the answers to these questions:

- Are all the stakeholders ready for the delivery into the user community?

- Is the user satisfied?
