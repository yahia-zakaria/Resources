Role Descriptor: Project Manager (PM)
This role plans, manages and allocates resources, shapes priorities, coordinates interactions with customers and users, and keeps the project team focused. The Software Project Manager also establishes a set of practices that ensure the integrity and quality of project work products.

A Project Manager applies project management knowledge, skills, tools and techniques to a broad range of tasks in order to meet requirements and to deliver an end result for a particular project. The responsibilities of the Software Project Manager can be detailed as follows:

 1.       Implementing the software project using the standard PIM process assets and the measurement repository for estimating and planning project activities.

2.       Establishing and maintaining the project management’s work environment based on the PIM’s work environment standards.

3.      Integrating and managing the software project plan and other plans that affect the project to describe the project’s defined process.

4.      Establishing and maintaining project technical working teams.

5.      Contributing process related experiences to the PIM process assets.

6.      Managing the involvement of relevant stakeholders in the software projects.

7.      Participating with relevant stakeholders to identify, negotiate and track critical dependencies and resolve coordination issues.

8.      Estimating project planning parameters and work product attributes using work breakdown structure (WBS) techniques and the standard WBS Policy.

9.      Establishing and maintaining the software project’s effort and schedule.

10.   Monitoring actual values of project planning parameters against the project plan.

11.    Monitoring project commitments against those identified in the project plan.

12.    Periodically reviewing the project’s progress, performance, and issues.

13.   Reviewing the project’s accomplishments and results at the pre-defined project milestones.

14.   Collecting and analyzing issues and determine corrective actions to address them.

15.   Managing changes and change requests according to the standard change management process.

16.   Analyzing, evaluating and categorizing potential risks using the standard risk management policy.

17.   Developing risk mitigation plans in accordance with the risk management policy.

18.   Monitoring the status of each risk periodically and implement the risk mitigation plan as appropriate.
