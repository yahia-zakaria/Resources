Main Description

Creating a software product is also known as doing a ‘Build’. A build is done when a logical milestone is achieved. This milestone relates to the completion of a certain task or a set of tasks. Builds are of two types:

1)      Complete – A build is done taking into consideration all the configurable items from a baseline.

2)      Patch – A build is done for only those configurable items that have changed.

Builds are created and moved by designated personnel to the required environments (e.g. between development and Test Environments).

 

Builds shall be numbered in accordance with the convention mentioned in Software Configuration Management Policy. Promotion of builds to Integration and Pre-Production environments is done by the CM department using the installation instructions and provided by the project team and tool specific guidelines. 
Steps
Step 1. Send email to the Deployment engineer to prepare the tesing enviroment
Step 2. Check readiness confirmation of the system and tool
Step 3. Move build from dev branch to main branch on TFS
Step 4. Create draft Deployment Document
