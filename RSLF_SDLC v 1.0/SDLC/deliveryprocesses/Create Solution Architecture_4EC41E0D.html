Main Description
Steps
Step 1. Conduct meetings to discuss the business document and how we understand it.
Step 2. Define the technical requirements, principles and models that guide the solution architect.
Step 3. Define the reusable components needed and suggested components to be reusable.
Step 4. Define system components, tools and technologies, deployment model and high-level specifications
Step 5. Prepare the solution architecture document and link it with the requirements.
Step 6. Publishing the solution architecture document according to last published version to be reviewed.
