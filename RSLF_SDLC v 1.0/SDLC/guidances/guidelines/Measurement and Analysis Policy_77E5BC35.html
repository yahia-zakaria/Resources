Guideline: Measurement and Analysis Policy
The purpose of the Measurement and Analysis policy is to provide the principles behind the SDLC metrics and the purposes for which measurement is done and specify the list of the indicators within the indicator category.

Main Description

Measurement Objectives:

 

1. Control the time and delivery date of projects and avoid delays in the project releases.

2. Increase the efficiency of project resources and control effort consumption.

3. Improve planning efficiency and achieve more accurate estimation.

4. Increase the Effectiveness of project resources.

 

1. Project Tracking Measures

The following measures are used to monitor and control the project progress. Project tracking measures are collected and stored during the project as per the monitoring and control policy.

 

1. Effort Variance:

 

Objectives Covered: Objective  2 

This measure calculates the effort variance as a decimal value in project server. Tasks are created as per the project schedule and assigned to resources for completion.

 

EV % = ((Baseline work - work )/Baseline work)*100


 

Where:

 

All Units are in hours.

 

And Actual Effort is the effort recorded by the assigned resource on the task work item. This can be tracked at a Task, Phase and Project level. Default value would be at a project level

 

Measure Analysis:

Effort variance should not exceed 30 or -30. Variances beyond this have to be reported to the higher level management and might require re-planning.

 collect every tow weeks by PMs  and saved in status report

2. Schedule Variance:

 

Objectives Covered: Objective 1 

 

This measure calculates the schedule variance in decimal as expected at the end of the Task/Phase/Project. Variance is calculated based on the variance of the Task Finish Date divided by the task duration. All values are in days. Calculated in decimal in the mpp.

 

SV % =   (( Baseline Duration - Duration)/ Baseline Duration) *100                  

 

 

This can be tracked at a Task, Phase and Project level. Default value would be at a project level

 

Measure Analysis:

 

should not exceed 20 or -20. Variances beyond this have to be reported to the higher level management and might require re-planning.

These Metrix will be tracked in the MPP used by the project.


 collect every tow weeks by PMs and saved in status report

5)Resource Productivity Rate

 

Measure

Resource Productivity Rate

Objectives Covered

Objective 3

Objective

This measure shows the average hours of effort per function point. This represents the effort needed by NIC team to deliver one function point. This measure is the basis for planning (estimating) the effort needed for any new typical project. The measure is calculated as average of the productivity rate of all NIC typical projects completed so far.

Equation

Productivity Rate = Project Effort (In Man Hours) / Project Size (In Function Points)


Threshold

Difference between actual & estimate effort should not exceed 20%

Responsible Role

Process Engineer

Storage Place

shared Folder

Period

Project Closure
