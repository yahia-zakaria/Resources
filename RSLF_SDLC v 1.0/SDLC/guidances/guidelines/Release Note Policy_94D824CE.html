Guideline: Release Note Policy
The purpose of the release note policy is to help the SDLC and Technical team leader to create and fill the release note

Main Description

-After the design Phase Closing, meeting must be conduct between project manager and technical team leader  and solution architect , to discuss the design and Agree on the number of releases .

- when Team leader  ready for release he must create deployment document and release document and move the code from DEV to Main for  testing and build the code in the Main and deploy  it on the Test Environment.

- After fixing the bugs on Main and Dev, he merge Dev and Main and build from Main.

- When he need to send the current version to client he merge the Main to Production and create the Release document and update the Deployment.

- Team Leader Conduct Meeting with Deployment Engineer to discuss the Deployment and Release document.

-After the Deployment Engineer Deploy on the client location, he update the deployment and add the bugs he face in the TFS and Assign it to the team leader.
