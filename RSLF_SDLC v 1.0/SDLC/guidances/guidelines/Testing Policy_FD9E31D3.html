Main Description

Definition of Testing

 Testing is a process of verifying that the system meets all business requirements including functional and non-functional requirements (performance, reliability, security, usability). This verification is done to ensure that we are building the system right. In addition, testing validating whether the system will meet user and others stakeholder needs in its operational environment(s).

Objectives of Testing

•            Prevent defects by evaluate work products such as requirements, user stories.

•            Verify whether all specified requirements have been fulfilled

•            Check whether the test object is complete and validate if it works as the users and other stakeholders expect

•            Build confidence in the level of quality of the test object

•            Find defects and failures thus reduce the level of risk of inadequate software quality

•            Provide information to stakeholders to allow them to make informed decisions, especially regarding the level of quality of the test object

The key responsibilities of testing

•            Review and contribute to test plans

•            Analyze, review, and assess requirements, user stories and acceptance criteria, specifications, and models for testability (i.e., the test basis)

•            Identify and document test conditions

•            Design and implement test cases and test procedures

•            Prepare and acquire test data

•            Execute tests, evaluate the results, and document deviations from expected results

•            Reporting and tracking defects in a bug tracking system

•            Use appropriate tools to facilitate the test process

•            Automate tests as needed (may be supported by a developer or a test automation expert)

•            Evaluate non-functional characteristics such as performance efficiency, reliability, usability, security, compatibility, and portability (may be supported by non-functional expert)

•            Review tests developed by others

•            Collaborate with teams to enhance software quality delivered

 Basic views regarding testing and the testing profession

A Software Testing career involves working towards the assurance of software quality. More than 40% of the time and effort of software development is spent in various forms of testing involving various people like developers, testers, users, etc.

 

 Quality Level to be achieved:

                              • The application meets all the requirements.

                              • The test coverage is 100%.

                              • All the application functions and features are successfully implemented with high standard as agreed.

                              • No software bugs of high or medium severity and of high or medium priority are discovered in live environment.

                             

The level of independence of the test organization

Test organization have full independence, testing is performed by test experts from quality control section, they not involved in development team.

•                           • Test Analysis and Design ➔Test condition, Test data, Test case, Test set(optional)

•                           • Test implementation and execution ➔ Test execution, Report defect

•                           • Evaluating exit criteria and Reporting ➔ Confirmation testing (all exit criteria have been met)

•                           • Test closure activities ➔ close user story

 

Approach to Regression Testing

Use regression testing to make sure that tested software continues to perform after every modification. a full or partial selection of already executed test cases which are re-executed to ensure existing functionalities work fine.

Given below are the various steps involved in performing this Testing

•                           • Prepare a Test suite for Regression

•                           • Automate all the test cases of the test suite.

•                           • Update the Regression suite whenever it is required

•                           • Execute the Regression test cases whenever there is any change in the code, the bug is fixed, new functionality is added, an enhancement to the existing functionality is done, etc.

•                           • Create a test execution Report which includes the Pass/Fails status of the executed test cases.

 

Approach to Automation Testing

Begin with automate the regression testing following these steps:

•                           • Selecting Test Automation Framework

•                           • Selecting tool

•                           • Create and run the tests

•                           • Maintain scripts

 

Entry and Exit Criteria

Typical entry criteria include:

•                           • Availability of testable requirements, user stories

•                           • Availability of test environment.

•                           • Availability of necessary test tools.

•                           • Availability of test data and other necessary resources

 

Typical exit criteria include:

•                           • Planned tests have been executed.

•                           • A defined level of coverage (e.g., of requirements, user stories, acceptance criteria, risks, code) has been achieved.

•                           • The number of unresolved defects is within an agreed limit.

•                           • The number of estimated remaining defects is sufficiently medium or low.
