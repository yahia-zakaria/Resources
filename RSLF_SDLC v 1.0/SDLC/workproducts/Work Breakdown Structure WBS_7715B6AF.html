Artifact: Work Breakdown Structure (WBS)
The WBS organizes and defines the total scope of the software project. It subdivides the software project work into smaller, more manageable pieces of work based on the standard SDLC pattern.
Work Product Kinds: Project Management
Purpose

The WBS represents a clear description of the software project’s deliverables and scope; the ‘‘what’’ of the project. It is not a description of the SDLC or schedule that defines how or when the deliverables will be produced, but rather is specifically limited to describing and detailing the software project’s outcome or scope.
