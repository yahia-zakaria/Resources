Description
Main Description

The development subsystem is the physical analogue of the software design. Development and the development subsystems are initially defined in the implementation view in the  Software Architecture Document , and so are of primary importance at development time.
Tailoring
Representation Options

UML Representation: Package in the implementation view, either its top-level package, or stereotyped as <<development subsystem>>.

 

It is recommended that software architect uses development subsystems to decide how to map packages in design to subsystems and directories in implementation. Software architect has to decide how many levels of subsystems the project needs.
