Artifact: Build
This artifact produces an operational version of a system or part of a system that demonstrates a subset of the capabilities to be provided in the final product. A build comprises one or more development elements (often executable), each constructed from other elements, usually by linking of source code.
Work Product Kinds: Development
Purpose

The purpose of a build, constructed from other elements in the development, is to deliver a testable subset of the run-time functions and capabilities of the system. Builds can be constructed at all levels of a system, encompassing single or multiple subsystems.

 

Note that informal (internal) builds can be constructed by a developer for several reasons - unit testing, for example - using elements from the developer's private development workspace and the subsystem and system integration workspaces, as appropriate.
