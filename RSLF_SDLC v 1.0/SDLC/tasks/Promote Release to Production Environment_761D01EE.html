Task: Promote Release to Production Environment
This describes promoting the compiled source code which has been Pre-Production tested and approved by the test manager from the Pre-Production environment to the Production Environment.
Disciplines: Configuration Management
Relationships
RolesPrimary Performer:
  • Deployment and Support Engineer

    		Optional:
    • None
    Outputs
    • Deployment Document
    • Product
      Main Description

      A build selected to be promoted (by virtue of it having passed the System Integration and Pre-Production tests) to production environment is called a product. The successful movement of a release to production signifies the achievement of objective for a particular iteration/change/project.
      Steps
      Step 1. Confirmed all deployments on Staging First
      Step 2. Create TFS Task include all the details for the deployment
      Step 3. Check the file format as was mentioned in the policy file, do not accept any deployment files
      Step 4. Mention the deployment time by email and share it with all the members.
      Step 5. Packaging the Product by moving from the staging branch to production branch.
      Step 6. Deliver and Install the released build in the Production Environment.
      Step 7. Confirmed all deployments on Production and Close the TFS task
      Step 8. Update the draft deployment document
