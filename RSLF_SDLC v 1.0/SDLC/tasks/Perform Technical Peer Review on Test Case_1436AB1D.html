Task: Perform Technical Peer Review on Test Case
This task is designed to verify and examine selected work products in order to evaluate its technical content and quality.
Disciplines: Testing (QC)
Purpose

The purpose of a peer review is to provide a disciplined engineering practice for detecting and correcting defects in software artifacts, and preventing their leakage into field operations according to the Capability Maturity Model.
Relationships
RolesPrimary Performer:
  • Test Team leader (QC)
Additional Performers:
  • Test Manager (QC)
InputsMandatory:
  • Software Test Plan
  • Test Cases
    		Optional:
    • None
    Outputs
    • Peer Reviews Document
      Main Description

      This task to review and validate if the test cases are created correctly and cover all business requirements.

      Steps
      Step 1. Identify Work Products to be Reviewed
      Step 2. Prepare for the Review
      Step 3. Select a Peer Review Type (Meeting or Offline)
      Step 4. Conduct the Technical Peer Review
      Step 5. Record Review Results and Comments
      Step 6. Finalize the Review and Update Status
