Task: Perform System Test
This task describes how to create and execute a set of tests to validate that the system or subsystem is working properly before more formal integration and preproduction testing are performed on the system.
Purpose

The purpose of system test is to implement set of tests that enable the validation of the entire system or subsystem through physical execution and to develop tests that can be executed in conjunction with other tests as part of a larger test infrastructure.

        Main Description

        In a typical software development project, ‘unit testing’ is done by the Developers. This ensures that the individual components are working fine. The ‘Integration testing’ focuses on successful integration of all the individual pieces of software (components or units of code). Once the components are integrated and tested, the system as a whole needs to be rigorously tested to ensure that it meets the functional and technical requirements before being delivered to client for integration with its other systems and acceptance.

        The purpose of System testing is to demonstrate that the integrated product meets the defined functional and technical requirements and is carried out in the development environment by the designated Developers. Completion of system testing is a pre requisite for System Integration Testing.

        Some or all of the following elements are examined as part of System Testing :

        1. Functionality Testing

        2. Compatibility / Conversion Testing

        3. Configuration Testing

        4. Documentation Testing

        5. Installation Testing

        6. Performance Testing

        7. Procedure Testing

        8. Recovery Testing

        9. Reliability Testing

        10 .Security Testing

        11. Serviceability Testing

        12. Storage Testing

        13. Usability Testing
